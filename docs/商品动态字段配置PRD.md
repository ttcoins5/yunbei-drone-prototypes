# 商品动态字段配置 PRD

## 1. 背景与目标

客户最新提供的服务信息采集表显示，不同无人机服务需要填写的字段差异明显。后续商品不再配置独立的预约开关；联系人、联系方式作为下单默认必填项，其余具体填写内容全部由后台“需求采集字段”配置决定，以覆盖巡检、物流、吊运、表演、托管等服务。

本次新增“商品需求采集字段”能力，后台运营在每个商品上单独维护下单字段；小程序下单页按商品配置动态展示字段；订单生成后保存字段快照，后台、用户端、飞手端均按快照查看。

## 2. 功能范围

### 2.1 后台商品编辑

- 商品编辑页新增“需求采集字段”模块。
- 支持新增字段、删除字段、上移、下移。
- 字段配置项包括：字段名称、字段类型、是否必填、提示文案、下拉选项。
- 字段类型首版只支持：文本、下拉框、图片。

### 2.2 小程序下单

- 商品详情页保持现有结构。
- 用户点击“立即下单”进入确认订单页。
- 确认订单页展示“需求信息”，字段来自当前商品的独立字段配置。
- 联系人、联系电话为默认必填字段，优先带出用户资料；其他文本、下拉、图片字段按商品配置展示。
- 必填字段为空时阻止提交，并提示“请填写字段名”。
- 图片字段展示上传入口，提交后订单快照保存文件名。

### 2.3 订单快照

- 订单提交时保存 `requirementSnapshot`。
- 快照包含商品字段配置名称、字段名称、字段类型、用户填写值、是否必填。
- 商品后续修改字段配置，不影响历史订单展示。
- 后台订单详情展示“需求信息快照”。
- 用户订单详情展示“需求信息”。
- 飞手分配订单详情展示履约所需的动态字段。

## 3. 不做范围

- 不做字段联动，例如选择客户类型后动态展示不同字段。
- 不做复杂公式、自动报价、条件校验。
- 不做按字段统计分析。
- 不开放客户自行配置全平台表单规则。
- 不为每个服务单独开发独立下单页面。

## 5. 页面与交互

### 5.1 商品编辑页

1. 运营进入后台商品管理，点击编辑商品。
2. 在“需求采集字段”模块直接维护当前商品字段。
3. 运营可新增字段、修改字段名称、字段类型、必填状态、提示文案、下拉选项、排序。
4. 点击“保存商品”后生效。

### 5.2 小程序下单页

1. 用户进入商品详情页。
2. 点击“立即下单”。
3. 系统根据当前商品字段配置渲染“需求信息”表单。
4. 用户填写并提交。
5. 系统校验必填字段。
6. 校验通过后生成待支付/待接单订单，并保存字段快照。

### 5.3 订单详情

1. 后台查看订单详情时，展示订单状态、订单信息快照和需求信息快照。
2. 用户查看订单详情时，展示自己提交的需求信息。
3. 飞手查看分配订单详情时，展示与履约相关的需求信息。

## 6. 验收标准

- 为巡检、物流、吊运、表演、托管分别创建商品并配置独立字段，字段能正常展示。
- 修改某个商品字段，不影响其他商品字段。
- 修改商品字段顺序后，小程序下单页顺序同步变化。
- 必填字段为空时不能提交订单。
- 图片字段能展示上传入口，订单详情能看到文件名。
- 商品字段修改后，历史订单仍展示下单时的字段快照。
- 后台订单详情、用户订单详情、飞手订单详情均可展示动态需求信息。

## 7. 建议默认值

- 首版采用 JSON 动态字段保存，不新增大量业务字段列。
- 字段配置只开放给后台运营，不开放给普通用户。
- 每个商品独立维护字段，不提供套用入口。
- 外卖配送若只是跳转第三方或独立业务页，首版不纳入通用商品表单。
