## 四、需求背景

### 4.1 目标用户现状

| 用户角色 | 使用场景 | 当前要完成的事 |
|---|---|---|
| 普通用户 | 微信小程序内浏览无人机服务、培训、租赁、外卖跳转等入口 | 找到合适服务，提交需求，查看订单、发票、消息和个人资料 |
| 认证飞手 | 在同一个小程序中完成入驻认证、查看任务和分配订单 | 提交资质，承接平台任务，查看派单订单，提交飞行报备 |
| 后台运营/管理员 | Web 后台内维护配置、商品、订单、飞手、任务、发票和运营信息 | 配置小程序内容，处理订单和审核，查看业务线索 |

### 4.2 核心痛点

| 问题点 | 问题描述 | 当前影响 |
|---|---|---|
| 跨端口径分散 | 后台、小程序普通用户端、飞手端各自有页面和 mock 数据 | 后续维护容易出现字段、状态、入口不一致 |
| 商品服务差异大 | 巡检、吊运、物流、表演、托管、培训、赛事、外卖等信息结构不同 | 需要动态字段和详情模板承接差异，不适合写死所有页面 |
| 履约链路涉及多角色 | 用户下单、后台派单、飞手承接、用户查看进度互相关联 | 订单状态、飞手状态、需求快照必须统一 |
| 当前仅为静态原型 | 没有真实后端、账号、支付、文件、消息、第三方推送 | 后续接口和数据库设计需要明确原型当前边界 |

### 4.3 需求目标

| 目标 | 成功标准 |
|---|---|
| 建立统一需求基线 | 后台、小程序普通用户端、飞手端功能均可在本文档找到入口、字段、状态和跨端关系 |
| 支撑后续原型维护 | 每次原型调整都有明确 PRD 更新位置和变更记录 |
| 支撑接口和数据模型设计 | 订单、商品、用户、飞手、任务、发票等核心对象字段口径统一 |
| 降低评审成本 | 产品、设计、研发、测试可按本文档逐模块评审和拆分任务 |

## 八、核心功能说明

### 8.1 商品服务目录与详情模板

**功能描述**

后台运营维护分类、商品、规格、业务属性、需求字段和详情页模板；小程序按上架商品展示服务内容，并引导用户下单、报名或跳转外部服务。

**用户流程**

```text
后台商品管理
  ↓
维护分类、商品、规格、业务属性、需求字段、详情模板
  ├─ 保存成功 → 小程序首页/分类/商品列表消费展示
  ├─ 分类下有商品 → 禁止删除分类
  ├─ 商品有关联订单 → 禁止删除商品
  └─ 保存失败 → 保留编辑内容并提示重试
      ↓
用户进入小程序商品详情
  ↓
按模板查看服务/培训/赛事/外部跳转内容
  ├─ 服务类 → 立即下单
  ├─ 培训/赛事类 → 立即报名或提交信息
  ├─ 外部跳转类 → 跳转第三方
  └─ 商品下架/配置缺失 → 展示不可用或默认内容
```

**状态机**

| 状态名称 | 进入条件 | 退出条件 | 用户可见表现 |
|---|---|---|---|
| 草稿/编辑中 | 新建或编辑商品 | 保存成功 | 后台显示编辑态 |
| 上架 | 商品状态启用 | 下架或删除 | 小程序可见并可进入详情 |
| 下架 | 商品状态关闭 | 重新上架 | 小程序不展示或提示暂不可下单 |
| 有订单关联 | 商品生成过订单 | 无退出条件 | 后台禁止删除 |
| 配置异常 | 模板或字段缺失 | 补齐配置 | 小程序使用默认模板或提示不可用 |

**字段规范**

| 字段名 | 类型 | 是否必填 | 长度限制 | 校验规则 | 错误提示 |
|---|---|---|---|---|---|
| 商品名称 | 文本 | 是 | 40 字 | 不可为空 | 请填写商品名称 |
| 商品分类 | 枚举 | 是 | 固定分类 | 必须选择启用分类 | 请选择商品分类 |
| 上架状态 | 布尔 | 是 | true/false | 默认上架 | 商品状态无效 |
| 商品图标 | 图片 | 是 | 单张 | JPG/PNG | 请上传商品图标 |
| 规格名称 | 文本 | 是 | 30 字 | 至少 1 个规格 | 请添加规格 |
| 规格价格 | 数字 | 是 | 0-999999 | 不可为负数 | 请输入有效价格 |
| 在线支付 | 布尔 | 是 | true/false | 默认按商品配置 | 支付属性无效 |
| 是否需要飞手 | 布尔 | 是 | true/false | 默认按商品配置 | 飞手属性无效 |
| 需求字段名称 | 文本 | 是 | 20 字 | 启用字段不可为空 | 请填写字段名称 |
| 需求字段类型 | 枚举 | 是 | text/select/image 等 | 必须为支持类型 | 请选择字段类型 |
| 详情模板 | 枚举 | 是 | service/training/event/external | 必须选择 | 请选择详情页模板 |

**文案规范**

| 场景 | 文案内容 |
|---|---|
| 空状态 | 暂无可用服务 |
| 加载中 | 正在加载服务信息... |
| 操作成功 | 商品信息已保存 |
| 操作失败 | 商品信息保存失败，请重试 |
| 权限不足 | 当前账号无商品编辑权限 |
| 网络异常 | 网络异常，服务信息暂未更新 |

**异常情况处理**

| 异常场景 | 触发条件 | 处理方式 | 用户反馈 |
|---|---|---|---|
| 商品下架 | 用户访问已下架商品 | 禁止下单，保留返回入口 | 该服务暂不可下单 |
| 字段未配置 | 商品无需求字段 | 仅展示默认联系人和手机号 | 请完善联系信息 |
| 图片加载失败 | 图片资源不可访问 | 使用占位图 | 图片加载失败 |
| 外部链接失败 | 第三方链接无法打开 | 留在当前页，提供客服入口 | 暂无法打开配送服务 |

### 8.2 用户下单与订单履约

**功能描述**

用户在小程序选择商品和规格，填写联系人、手机号及商品动态需求字段；系统生成订单并保存商品快照、需求快照。后台接收订单后按商品属性处理支付、派单和履约状态。

**用户流程**

```text
商品详情
  ↓
点击立即下单
  ↓
确认订单并填写需求信息
  ├─ 必填字段缺失 → 阻止提交并定位字段
```

## 九、跨端数据与状态规则

### 9.1 共用核心对象

| 对象 | 后台来源 | 小程序消费 | 说明 |
|---|---|---|---|
| 首页轮播 | 轮播图配置 | 首页视觉区 | 1 个视频固定首位，最多 5 张图片 |
| 首页导航 | 导航配置 | 首页 8 个入口 | 大图/小图组内排序，支持内链和外链 |
| 案例展示 | 案例配置 | 首页/案例页 | 用于展示服务案例和详情 |
| 商品分类 | 商品分类 | 分类页/商品列表 | 分类下有商品不可删除 |
| 商品 | 商品编辑 | 首页、分类、详情、下单 | 商品属性进入订单快照 |
| 需求字段 | 商品编辑 | 确认订单 | 下单后进入 `requirementSnapshot` |
| 订单 | 用户提交/后台处理 | 我的订单/飞手订单 | 建议统一订单主键为 `orderNo` |
| 用户资料 | 小程序授权/完善 | 后台用户管理 | 后台只读查看 |
| 飞手资料 | 小程序提交/后台审核 | 飞手工作台、派单 | 通过后普通用户具备飞手身份 |
| 任务需求 | 后台发布 | 飞手任务大厅 | 收集意愿，不自动转订单 |
| 飞行报备 | 飞手提交 | 后台确认 | 第三方推送为预留 |
| 发票申请 | 用户提交 | 后台处理 | 支持个人普票和企业普票 |

### 9.2 订单状态默认规则

| 支付方式 | 是否需要飞手 | 状态链路 |
|---|---|---|
| 在线支付 | 需要飞手 | 订单生成 -> 待付款 -> 待派单 -> 待服务 -> 待评价 -> 已完成 |
| 在线支付 | 无需飞手 | 订单生成 -> 待付款 -> 待交付 -> 待评价 -> 已完成 |
| 非在线支付 | 需要飞手 | 订单生成 -> 待派单 -> 待服务 -> 待评价 -> 已完成 |
| 非在线支付 | 无需飞手 | 订单生成 -> 待交付 -> 待评价 -> 已完成 |

### 9.3 关键建议默认值

| 编号 | 规则 | 建议默认值 |
|---|---|---|
| 1 | 订单编号 | 统一使用 `orderNo` 作为跨端展示和关联主键 |
| 2 | 飞手接单 | 首版由后台分配后直接生效，飞手端只读查看 |
| 3 | 飞手履约更新 | 首版不开放飞手主动改状态，后续再做开始服务/完成服务/成果上传 |
| 4 | 任务意愿取消 | 征集中可取消，任务关闭后不可取消 |
| 5 | 任务与订单关系 | 任务大厅只收集意愿，不自动生成商品订单 |
| 6 | 发票类型 | 首版支持个人普票和企业普票，不做专票 |
| 7 | 运营商申请/反馈 | 首版按信息收集处理，不做审核和回复闭环 |
| 8 | 地址簿 | 用户侧私有，后台只在订单快照中查看服务地址 |
