# 系统权限
## 路由与代码位置
- 路由：`admin-users`、`roles`
- 代码：`js/modules/access-control.js`
## 当前功能
- 后台用户列表、新增/编辑用户、分配角色、启停状态展示。
- 角色列表、新增/编辑角色、按页面勾选权限。
- 支持“切换视角”模拟不同角色登录后的左侧菜单可见范围。
## 数据结构
- `adminUsers[]`、`adminRoles[]`、`permissionCatalog[]`。
- `state.currentAdminUserId` 记录当前模拟登录的后台用户。
## 操作与业务规则
- 超级管理员为系统内置角色，默认拥有全部页面权限，不允许修改。
- 被后台用户占用的角色不可删除。
- 当前角色无权访问某路由时，自动跳转到该角色首个可访问页面。
## 与其他模块的依赖
- 依赖 `core/router.js` 的菜单过滤和路由拦截。
- 页面权限按一级页面路由控制，详情页通过 `navRouteAlias` 归属到对应列表权限。
## 已知占位行为
- 登录仍为原型模拟；停用用户仅在切换视角时拦截，真实系统应在登录阶段拦截。
