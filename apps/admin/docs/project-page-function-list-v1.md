**商品规则**

- 分类下存在商品时不可删除分类。
- 商品存在关联订单时不可删除商品。
- 商品轮播图最多 9 张，第一张作为列表封面。
- 商品业务属性在订单生成时保存为订单快照，后续修改商品不影响历史订单。

### 4. 订单管理

| 页面 | 主要字段/内容 | 页面功能 | 状态 |
|---|---|---|---|
| 订单列表 | 订单号、用户、商品/服务、金额、是否需要飞手、状态 | 按订单/用户/商品/状态/飞手需求筛选、分页、查看详情、待派单订单直接派单 | 已有 |
| 订单详情 | 状态流程、订单快照、需求信息快照、联系电话、服务地址、说明、附件、已分配飞手 | 预览附件、分配飞手、调整飞手、查看履约状态 | 已有 |

**订单状态规则**

- 在线支付 + 需要飞手：订单生成 → 待付款 → 待派单 → 待服务 → 待评价 → 已完成。
- 在线支付 + 无需飞手：订单生成 → 待付款 → 待交付 → 待评价 → 已完成。
- 非在线支付 + 需要飞手：订单生成 → 待派单 → 待服务 → 待评价 → 已完成。
- 非在线支付 + 无需飞手：订单生成 → 待交付 → 待评价 → 已完成。
- 待派单确认分配后进入待服务；待服务阶段调整飞手不改变订单状态。

## 五、业务规则确认表

修改“建议默认方案”或在“最终结论”中填写确定结果。

| 编号 | 待确认问题 | 建议默认方案 | 最终结论 |

| 2 | 飞手收到派单后是否接单/拒单？ | 首版沿用后台规则，管理员分配后直接生效 | 待填写 |

## 六、建议评审顺序

1. 先确认商品下单、支付、需求字段和订单状态。
2. 再确认飞手派单、履约和任务意愿规则。
3. 确认普通用户与飞手端页面范围。
4. 确认培训、发票、评价和飞行报备的第二阶段范围。
5. 最后确认品牌素材、页面文案和移动端视觉风格。

清单确认后，再初始化独立的 `drone-miniapp-prototype` 微信原生小程序项目。
